# 5G CPE 怎么选？映翰通 CPE02 室内 5G 路由器的组网思路与部署要点

> 本文基于映翰通 CPE02 产品规格书整理，面向需要做**快速部署、高速无线接入、多点位远程运维**的读者，侧重技术选型与组网实践，不构成采购承诺。

## 1. 背景：什么场景会考虑 5G CPE？

在以下几类需求里，固定光纤/ADSL 往往不是最优解：

| 场景特征 | 典型例子 | 常见痛点 |
|----------|----------|----------|
| 施工周期长、改造成本高 | 精装房、老小区、商铺 | 拉线破坏装修，开通等待久 |
| 点位分散、现场无人 | 连锁门店、民宿、快闪 | 本地运维成本高，故障响应慢 |
| 周期短、可迁移 | 展会、项目部、临时办公 | 专线不划算，4G 带宽不足 |
| 需要备份链路 | 收银/POS、监控、视频会议 | 单宽带故障即业务中断 |

5G CPE（Customer Premises Equipment）的核心价值，是把**蜂窝网络**转成**Wi-Fi + 有线 LAN**，在室内快速提供接入能力。选型时除了看峰值速率，还要关注：**并发能力、双链路备份、云管、防护等级、安装形态**。

映翰通 **CPE02** 定位是**室内桌面/壁挂式 5G CPE**，与 IR 系列工业路由（导轨、宽温、IP30 柜内）形成互补。下面按「产品能力 → 组网 → 部署 → 选型边界」展开。

## 2. CPE02 产品概览

### 2.1 形态与物理参数

CPE02 采用白色立式「小塔」设计，主要机械参数如下：

| 参数 | 规格 |
|------|------|
| 尺寸 (长×宽×高) | **105 × 105 × 217 mm** |
| 重量 | 800 g |
| 安装方式 | 桌面式、壁挂式（支架可选） |
| 防护等级 | IP20（室内） |
| 工作温度 | -10 °C ~ +40 °C |
| 供电 | 12 V / 2 A，功耗 ≤ 15 W |

> **关于「105 mm」**：规格书与尺寸图（`images/media_1_5.png`）标注的 105.00 为**底座边长**（长、宽均为 105 mm），高度为 217 mm。标题/marketing 语境里常简写为「105 mm 见方底座的小白塔」，完整尺寸应写 **105×105×217 mm**。

![CPE02 产品正视图](images/media_1_13.png)

*图 1：CPE02 立式外观（建议上传 CSDN 图床后替换链接）*

### 2.2 核心能力一览

| 能力域 | 要点 |
|--------|------|
| **蜂窝** | 5G SA/NSA + 4G LTE；NSA 下行最高 2.6 Gbps，SA 下行 2.0 Gbps；4×4 MIMO Sub-6 |
| **Wi-Fi** | Wi-Fi 6 双频，中国版 AX3000（574 + 2402 Mbps）；推荐 Wi-Fi 并发 128 台 |
| **有线** | 中国版 2×GbE，支持 WAN/LAN、双 LAN |
| **链路** | 5G/4G + 有线宽带双接入；自动故障切换、负载均衡、链路探测 |
| **云管** | 小星云管家：零接触部署、批量升级、配置下发、流量/信号/客户端统计 |
| **安全** | 防火墙 1 Gbps；IPsec/L2TP/OpenVPN*；WPA2/WPA3*；SHA512 管理密码 |
| **SIM** | 1× Nano 4FF；eSIM 可选 |

CPU 为 **IPQ5018**，防火墙吞吐 **1 Gbps**，推荐总用户数 **200**（其中 Wi-Fi 128）。对中国区项目，主流型号为 **CPE02-CNNR**。

### 2.3 型号与区域差异（节选）

| 型号 | 区域 | 以太网 | Wi-Fi |
|------|------|--------|-------|
| CPE02-CNNR | 中国 | 2 × GbE | AX3000 |
| CPE02-NANR | 北美 | 1 × 2.5G + 1 × 1G | AX5400 |
| CPE02-EUNR | 欧洲/亚太 | 2 × 1G | AX3000 |

5G/LTE 频段列表详见规格书第 5 章，部署前请按运营商与项目所在地核对 band 支持。

## 3. 典型组网拓扑

CPE02 常见有三种组网角色，可根据项目选择。

### 3.1 5G 主力接入（无固定宽带）

适合：新装商铺、民宿、临时办公、宽带无法及时开通的现场。

```
                    ┌─────────────┐
   5G 基站 ──蜂窝──▶│   CPE02     │
                    │  (5G CPE)   │
                    └───┬────┬────┘
                        │    │
                   Wi-Fi│    │GbE
                        │    │
              ┌─────────┴    └──────────┐
              ▼                         ▼
        手机/平板/IoT              PC/NVR/交换机
```

要点：
- SIM 卡 + 电源即可上线，适合「即插即用」交付。
- 建议将设备放置在**靠窗或信号较好**的位置，避免金属柜体深度遮挡。
- 高并发场景（>30 台同时在线）建议划分**多 SSID**（客用/内网分离）。

### 3.2 5G + 有线双链路备份（推荐轻商用）

适合：门店、SOHO、诊所等「有宽带但怕断线」的场景。

```
  光纤/PPPoE ──WAN──▶┌─────────────┐
                      │   CPE02     │◀── 5G/4G（备份或负载）
                      │ 双链路探测  │
                      └───┬────┬────┘
                          │    │
                     LAN1 │    │ Wi-Fi 6
                          ▼    ▼
                       POS/PC  顾客/员工终端
```

要点：
- 启用**链路探测 + 自动切换**，主链路故障时切到蜂窝，减少收银/监控中断时间。
- 可配置**负载均衡**分摊流量，但需注意蜂窝流量成本，建议对客用 Wi-Fi 做限速或 Portal*。
- 2 个 GbE 口有限，下挂设备多时应**外接千兆交换机**。

### 3.3 多点位 + 小星云管家（连锁/渠道）

适合：多门店、多民宿、分布式轻商用网络统一运维。

```
  ┌──────── CPE02 @ 门店 A ────────┐
  ├──────── CPE02 @ 门店 B ────────┤
  ├──────── CPE02 @ 民宿 C ────────├──▶ 小星云管家（配置/监控/告警/升级）
  └──────── CPE02 @ 快闪 D ────────┘
                    │
                    └──▶ 运维人员 Web / APP 统一视图
```

平台侧常见能力：
- 批量配置模板、计划固件升级
- 联网状态、延迟/丢包/吞吐、蜂窝 RSRP、流量统计
- 客户端分析、上行链路管理、邮件/短信告警

对集成商而言，价值在于把「每台设备本地调」变成「平台批量管」，尤其适合**点位多、现场进不去**的项目。

## 4. 部署实践要点

### 4.1 硬件安装 checklist

1. **选位**：室内、通风良好；避免密闭金属柜；壁挂时注意天线/壳体不要被大面积遮挡。
2. **供电**：使用原装 12 V/2 A 适配器；长期在线场景建议 UPS 或带电池插座。
3. **SIM**：Nano 4FF；批量项目可评估 **eSIM** 降低换卡成本。
4. **有线**：WAN 接宽带时确认 PPPoE 账号或 DHCP；LAN 接下位机或交换机。
5. **指示灯**：5G / Wi-Fi / Signal / Power，便于现场快速排障。

### 4.2 软件配置建议（逻辑顺序）

| 步骤 | 配置项 | 说明 |
|------|--------|------|
| 1 | 接入小星云管家 | 零接触上线，便于远程维护 |
| 2 | APN / 拨号 | 按运营商卡配置；启用蜂窝自动重拨 |
| 3 | WAN 与双链路策略 | 主备或负载；开启链路探测 |
| 4 | Wi-Fi SSID 规划 | 客用/员工/IoT 分 SSID；按需 VLAN |
| 5 | 安全策略 | 修改默认密码；防火墙规则；VPN 按需 |
| 6 | 告警 | 本地邮件 + 平台短信/邮件 |

Wi-Fi 侧支持：**多 SSID、SSID VLAN、隐藏 SSID、WPS 客户端一键连接**。轻商用建议关闭不必要的 WPS（若安全规范要求），改用手动 WPA2/WPA3* 配网。

### 4.3 性能预期（避免误解）

需要 realistic 预期：

- 规格书中的 **2.6 Gbps / 2.0 Gbps** 为蜂窝模组**理论峰值**，实际速率受信号、频段、运营商套餐、并发数影响。
- **AX3000** 为 Wi-Fi 空口协商速率，不等于每个终端都能跑满。
- 防火墙吞吐 **1 Gbps**，一般轻商用足够；大流量专线场景应评估更高端型号。
- 推荐 **128 Wi-Fi 并发 / 200 总用户** 为设计参考值，高密度场景应做现场压测。

## 5. 适用场景与边界

### 5.1 较适合的场景

| 场景 | 为什么适合 CPE02 |
|------|------------------|
| 家庭 / 公寓 | 底座 105×105 mm 占地小；AX3000 覆盖多终端；5G 作主力或备份 |
| 咖啡馆 / 零售门店 | 客用 Wi-Fi + 收银/监控；云管看多店状态 |
| 民宿 / 短租 | 即插即用；远程改 Wi-Fi 密码/重启 |
| SOHO / 小团队 | 2×GbE + Wi-Fi 6；视频会议、云盘同步 |
| 快闪 / 临时项目部 | 快速拉起网络，结束可迁移 |

### 5.2 不太适合 / 需换型的场景

| 需求 | 建议 |
|------|------|
| 导轨安装、配电箱、无人值守柜内 | 考虑 **IR302 / IR624** 等工业路由 |
| 宽温（如 -35 °C 户外） | CPE02 仅 -10~+40 °C，需户外/工业型号 |
| 高防护（淋雨、粉尘） | CPE02 为 **IP20 室内** |
| 多 LAN 口（>2 且无交换机） | 外接交换机或选多网口企业路由 |
| 双 SIM 硬冗余 | CPE02 为单 Nano SIM（eSIM 可选），双卡需求看 IR/ER 系列 |

写选型文档时，把边界说清楚，反而更容易建立技术信任。

## 7. 常见问题 FAQ

**Q1：CPE02 能否替代企业级光纤网关？**  
A：可作为**接入层 CPE** 或**备份链路**，但不等同于大型园区核心网关。多 VLAN、大流量策略路由*等需结合项目评估。

**Q2：只有 5G 没有宽带，延迟能开会吗？**  
A：5G 时延通常可满足一般视频会议，但取决于信号与运营商网络。建议关键业务保留有线主链路，5G 作备份更稳妥。

**Q3：128 台 Wi-Fi 并发是怎么理解的？**  
A：指推荐设计容量，不是保证每台同时满速率。高密度场景请实测，并做 SSID/频段规划。

**Q4：如何远程运维？**  
A：接入**小星云管家**后可远程配置、升级、看流量与信号；平台集成 Ookla 测速便于验收。

**Q5：中国区买哪个型号？**  
A：**CPE02-CNNR**，2×GbE + AX3000，频段见规格书订购表。

## 8. 总结

CPE02 适合需要**快速把 5G 变成 Wi-Fi + 有线**的室内场景：105×105×217 mm 立式形态、Wi-Fi 6 AX3000、5G/有线双链路、小星云管家云管，构成其核心价值。

选型时建议抓住四个问题：
1. 环境是否**室内 IP20** 可接受？
2. 需要**5G 主力**还是**宽带 + 5G 备份**？
3. 并发规模是否落在 **128 Wi-Fi / 200 用户** 设计范围内？
4. 是否有多点位**远程运维**需求？

若以上答案偏「是」，CPE02 值得纳入方案比选。更完整的频段、VPN、防火墙参数请参阅官方规格书。
